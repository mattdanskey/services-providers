Singletons, Singletons everywhere

Value

  • What you pass in is its value.
  • Can't use dependencies
  • If it's an object, its state is shared wherever it's injected.
angular.module('myApp', [])
.value('myValue', 98128)

Factory

  • Takes a function
  • Return value is singleton & is what is injected
angular.module('myApp')
.factory('myFactory', function(myValue) {

  return {
    foo: "Alberta",
    bar: function() {
      return myValue / 2;
    }
  };

});

Service

  • Takes a constructor function
  • Calls it with new
angular.module('myApp')
.service('foo', function(myFactory) {

  this.wat = myFactory.bar() + '';

  this.bar = function() {
    return this.wat.split('');
  }.bind(this);

});
angular.module('myApp')
.service('foo', function() {

    this.hello = "Hi!";

});
angular.module('myApp')
.factory('foo', function(){

  function foo() {
    this.hello = "Hi!";
  }

  return new foo();
});

Provider

  • Secretly all of the preceding service types!
  • Allows for alteration during the config stage
  • Only can depend on constants
  • this.$get
angular.module('myApp')
.provider('crayonColor', function() {
  this.hue = "red";

  this.$get = function(foo) {
    return {
      crayon: hue,
      listOfNumberStrings: foo.bar();
    };
  };
});
angular.module('myApp')
.config(function(crayonColorProvider){

crayonColorProvider.hue = "blue";

})

Constant

  • Not immutable (e.g. if an Object, its value can be changed)
  • Basically a value, except:
  • Can be injected into a provider
  • Can't be intercepted by a decorator
angular.module('myApp', [])
.constant('hands', 2.3)
